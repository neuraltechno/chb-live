## Implementation Plan - Current Round Games Banner

We will implement a horizontal banner in the header that displays the current round's games across all sports. This banner will provide quick access to live and upcoming matches directly from the navigation bar.

## Proposed Changes

### 1. Update `GameBanner` Component
- Modify `src/components/GameBanner.tsx` to accept `selectedRound` and `selectedSport` props.
- Pass these props from the `Navbar` (or derive them in the banner if we move logic there, but props are cleaner).
- Use the `convex/games.ts:list` query with the `round` and `sport` filters to fetch only the games for the current round.
- Remove the client-side slicing (`.slice(0, 9)`) since the API will now return the filtered set (5-9 games).
- Ensure date formatting uses 3-letter abbreviations (e.g., "Mon", "Tue").

### 2. Update `Navbar` and State Management
- Modify `src/components/Navbar.tsx` to access the `selectedRound` and `selectedSport` from the global store (`useGameStore`).
- Pass these values as props to the `GameBanner` component.
- Remove "Matches" link and "Live Sports Chat" text to make space.
- Ensure the layout is responsive (e.g., hiding or shrinking the banner on small screens).

### 3. Styling and Layout
- The banner will be integrated into the main `Navbar` height (`h-20`).
- Use a dense flex or grid layout to fit the games between the Logo and Auth Section.
- Ensure the banner handles the variable number of games (5-9) gracefully, perhaps using `flex-1` or a responsive grid.

### 4. Logic for "Current Round"
- The `useGames` hook already provides `currentRound`. We can use this value from the store or the hook to ensure the banner stays in sync with the home page filter.
- The `convex/games.ts:list` query supports a `round` argument. We will use this to ensure we are only showing the relevant matches.

## Implementation Steps

1. **Fix `GameBanner.tsx` duplicate code**:
   - Remove the duplicate `GameBannerItem` and `GameBanner` definitions (lines 105-194).

2. **Refactor `GameBanner`**:
   - Add `round` and `sport` props.
   - Update the `useQuery` call to include these filters.
   - Remove the `.slice(0, 9)` and rely on the API response size.

3. **Refactor `Navbar`**:
   - Import `useGameStore` to get `selectedRound` and `selectedSport`.
   - Remove the existing navigation links ("Matches", "Live Sports Chat").
   - Pass `selectedRound` and `selectedSport` to `<GameBanner />`.

4. **Adjust `RootLayout`**:
   - Ensure the main content padding (`pt-20`) matches the fixed header height.

5. **Verification**:
   - Test that changing the round on the home page updates the banner.
   - Verify links correctly navigate to match chat rooms.

## Questions / Tradeoffs
- **State Sync**: Should the banner update when the user changes the round on the home page? (Yes, using the global store ensures this reactivity).
- **Mobile View**: With 5-9 games in a single row, mobile screens will likely need to hide the banner or use a very compact view. I will keep it hidden on `xs` screens for now.

## Questions / Tradeoffs
- **Grid Layout**: With 9 games, a single row might be too wide for many screens. Should it be a 3x3 grid or a multi-column layout that wraps? (Initial implementation will aim for a dense, readable grid).
- **Mobile View**: Fitting 9 games without scrolling on mobile is difficult. Should we reduce the number of games displayed on mobile or use a smaller grid?
