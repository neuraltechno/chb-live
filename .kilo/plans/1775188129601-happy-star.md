# Implementation Plan - Current Round Games Banner

We will implement a horizontal banner in the header that displays the current round's games across all sports. This banner will provide quick access to live and upcoming matches directly from the navigation bar.

## Proposed Changes

### 1. New Component: `GameBanner`
- Create `src/components/GameBanner.tsx`.
- This component will fetch the current games using the `convex/games.ts:list` query.
- It will display a single row of up to 9 games.
- Each game card will show:
  - Two teams (Short names + logos if available)
  - Game status (Live badge with clock, Upcoming with time, or Finished)
  - Date (3-letter abbreviation, e.g., "Mon", "Tue") and Time
  - Venue (if available)
  - Link to the match page (`/match/[gameId]`)

### 2. Update `Navbar`
- Modify `src/components/Navbar.tsx` to include the `GameBanner` in the same horizontal row as the logo and auth buttons.
- Remove "Matches" link and "Live Sports Chat" text to make space.
- Ensure the layout is responsive (e.g., hiding or shrinking the banner on small screens).

### 3. Styling
- Use Tailwind CSS for a sleek, modern look.
- The banner will be integrated into the main `Navbar` height.
- Use a dense grid or flex layout to fit the games between the logo and auth section.

### 4. Logic for "Current Round"
- The `convex/games.ts:list` query already supports a 14-day window. We will limit the display to the 9 most relevant games (prioritizing Live, then Upcoming), and use 3-letter day abbreviations for all dates.

## Implementation Steps

1. **Update `GameBanner` component**:
   - Change date formatting to use `format(startDate, "EEE")`.
   - Simplify the component styling to fit inside the `Navbar` row.

2. **Refactor `Navbar`**:
   - Remove the existing navigation links ("Matches", "Live Sports Chat").
   - Move `GameBanner` from below the header to inside the main flex container between the Logo and Auth Section.
   - Adjust `RootLayout` (`src/app/layout.tsx`) padding back to `pt-16` since the header will be single-row again.

3. **Refine UI/UX**:
   - Ensure the banner handles overflow gracefully on smaller screens.
   - Maintain the "Live" pulsing effects.

4. **Verification**:
   - Test on desktop and mobile resolutions.
   - Verify links correctly navigate to match chat rooms.

## Questions / Tradeoffs
- **Grid Layout**: With 9 games, a single row might be too wide for many screens. Should it be a 3x3 grid or a multi-column layout that wraps? (Initial implementation will aim for a dense, readable grid).
- **Mobile View**: Fitting 9 games without scrolling on mobile is difficult. Should we reduce the number of games displayed on mobile or use a smaller grid?
